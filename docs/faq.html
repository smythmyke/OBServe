Answers to common questions about setup, licensing, and troubleshooting.

Getting Started

OBServe connects automatically via WebSocket. Just make sure OBS Studio is running with the WebSocket server enabled:

  1. Open OBS Studio
  2. Go to Tools → WebSocket Server Settings
  3. Check "Enable WebSocket Server"
  4. Set a port (default: 4455) and optionally a password
  5. Click OK, then launch OBServe

OBServe will detect OBS and connect within seconds. The connection status is shown in the system tray and on the dashboard.

OBServe requires OBS Studio 28.0 or later, which includes the built-in WebSocket v5 server. Earlier versions used a separate plugin (obs-websocket) with an incompatible protocol.

No. OBServe is built specifically for OBS Studio and relies on the OBS WebSocket v5 protocol. Streamlabs Desktop uses a different internal architecture and does not expose the same WebSocket API. We recommend using the free, open-source OBS Studio for the best experience.

  • OS: Windows 10 (64-bit) or later
  • RAM: 4 GB minimum (8 GB recommended)
  • Disk: ~15 MB for the app
  • OBS Studio: Version 28.0+ with WebSocket enabled
  • Internet: Required for AI features, license activation, and auto-updates

OBServe is built with Tauri + Rust for minimal resource usage — it typically uses less than 50 MB of RAM.

Licensing & Modules

After purchasing a module, you'll receive a license key on the confirmation page and via email.

  1. Open OBServe and go to the Store tab
  2. Paste your license key into the activation field
  3. Click Activate

The module will unlock immediately. Your key is saved locally so you won't need to re-enter it.

Each license key can be activated on up to 2 devices. This covers common scenarios like a desktop and a laptop. If you need to move your license to a new device, deactivate it on the old one first (Store tab → Deactivate), which frees up the device slot.

If you deactivate before reinstalling, the device slot is freed automatically. If you forgot to deactivate and have hit the 2-device limit, contact support with your purchase email and we'll reset your activation.

Use the Restore Purchases option in the Store tab. Enter the email address you used at checkout and your license keys will be sent to that email. You can also check your original purchase confirmation email.

Troubleshooting

  1. Confirm OBS Studio is running
  2. Check that the WebSocket server is enabled in Tools → WebSocket Server Settings
  3. Make sure the port matches (default: 4455)
  4. If you set a WebSocket password in OBS, enter the same password in OBServe's connection settings
  5. Check that your firewall isn't blocking localhost connections on port 4455

OBServe reads audio devices directly from Windows. Try these steps:

  • Make sure the device is plugged in and recognized by Windows (check Sound settings)
  • Restart OBServe — devices are detected on launch
  • If a device was added while OBServe is running, it should be detected automatically via hotplug. If not, restart the app.

The Video Editor module uses FFmpeg for export, remux, and caption burn-in. OBServe automatically detects FFmpeg from your OBS Studio installation (typically at C:\Program Files\obs-studio\bin\64bit\ffmpeg.exe).

If FFmpeg isn't found:

  • Make sure OBS Studio is installed in its default location
  • Alternatively, install FFmpeg separately and add it to your system PATH

OBServe checks for updates automatically on launch. When an update is available, you'll see a notification with the option to download and install. Updates are free and include all improvements to the base app and any modules you own.
